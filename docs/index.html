Usage Notes

This page is intentionally static and lightweight. It does not require a framework, build tool, database, or server-side runtime.

Best placement
Put this file in docs/index.html if your Pages source is the docs/ folder, or at the repo root if you publish from the root branch.
Why static HTML
A static landing page fits GitHub Pages naturally and avoids adding unnecessary server-side complexity for a repo that is already publishing files.
Why multi-path probing
The repo has both structured folders and older root-level artifacts, so the page checks multiple likely feed paths and picks the first one that responds successfully.
