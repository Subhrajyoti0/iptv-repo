Usage Notes

This dashboard is intentionally a single self-contained static HTML page. GitHub Pages is a static publishing workflow, and your repo already has a Pages-friendly `docs/` directory with `.nojekyll` and feed files there. [1](https://windowsreport.com/github-adds-rubber-duck-debugging-ai-to-copilot-cli-for-smarter-code-reviews/)[2](https://docs.github.com/copilot/how-tos/agents/copilot-coding-agent/using-copilot-to-work-on-an-issue)

Why static HTML
It avoids unnecessary server-side complexity and works naturally with GitHub Pages static hosting. [1](https://windowsreport.com/github-adds-rubber-duck-debugging-ai-to-copilot-cli-for-smarter-code-reviews/)
Why explicit feed and repo links
Your repo has both structured folders and older historical artifacts, so explicit links are more reliable than ambiguous auto-detection for the public landing page.
